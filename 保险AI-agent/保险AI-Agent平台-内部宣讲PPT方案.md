## 第1页：封面

**标题**：保险公司的知识蒸馏平台

**副标题**：让每个员工都拥有一个越用越聪明的AI分身

**视觉**：简洁大气，一个保险从业者的剪影 + 多个AI助手围绕在身边

## 第2页：一个真实的场景

**标题**：你有没有遇到过这种情况？

**内容**（用故事的方式）：

> 张姐是某保险公司资深核保员，干了8年，什么疑难case都见过。
>
> 去年张姐跳槽了。
>
> 她脑子里的判断经验、踩过的坑、对条款的理解——全部带走了。
>
> 新来的小李，要从头学起，至少半年才能独立核保。

**底部一行大字**：**人走了，知识也走了。这是保险公司最大的隐性损失。**

## 第3页：痛点不止这一个

**标题**：保险公司每天都在痛

**三个痛点并列**：

| 痛点 | 现状 |
|------|------|
| 🧠 知识流失 | 老员工离职，经验带走，公司什么都没留下 |
| 👶 培训成本高 | 新人培养周期长，一个核保员从入门到独立至少6个月 |
| 💬 重复劳动 | 客户群里同样的问题反复问，业务员疲于回答 |

**数据支撑**：
- 保险代理人从2019年高峰的**648万人**降至2024年的**264万人**，减少59%
- 人越来越少，活不会少，效率必须提上来

**视觉**：三个图标 + 关键词，简洁有力

---

## 第4页：市场有多大

**标题**：41万亿的行业，正在经历一场变革

**关键数据**：

| 指标 | 数据 |
|------|------|
| 保险业总资产 | **41.3万亿元**（2025年） |
| 年保费收入 | **6.1万亿元** |
| 从业人员 | **264万人**（还在减少） |
| 麦肯锡预测 | 核保自动化率将达**80%**，理赔超**40%** |

**底部结论**：
> 人在减少，业务在增长。保险公司**必须**用AI来补人效。
> 这不是"未来趋势"，是**现在正在发生的事**。

---

## 第5页：为什么是现在

**标题**：窗口期

**三个时间窗口并列**：

1. **AI Agent刚刚成熟** — 2025-2026年是Agent元年，能做复杂任务的AI刚出来
2. **保险行业在转型** — 从"人海战术"转向"精英化+数字化"，有钱有动力买
3. **还没有人真正做深** — 通用AI公司不懂保险，保险公司自己做不了AI

**底部一行大字**：**谁先做，谁吃掉飞轮红利。**

---

## 第6页：我们的产品

**标题**：给每个保险员工，发一个AI分身

**一张架构图**：

```
平台提供基础版Agent（核保、理赔、客服...）
         ↓
员工个性化定制（名字、头像、说话风格）
         ↓
Agent跟着员工，进入微信群工作
```

**关键设计**：
- 微信是入口 → 打开微信就能用，零学习成本
- 助手跟人走 → 业务员把Agent拉到任何群
- 一个群一个客户 → 隐私安全

## 第7页：具体怎么用（场景演示）

**标题**：一天的工作是这样的

**用模拟微信聊天截图展示两个场景**：

### 场景1：内部群
```
【核保讨论组】
业务员小王：@核保助手 这个客户有甲状腺结节3级，能标体承保吗？
核保助手：根据核保规则，甲状腺结节3级需要...
           [专业、准确、秒回]
```

### 场景2：客户群
```
【张三的服务群】
客户张三：@理赔助手 我昨天车被刮了，怎么报案？
理赔助手：张先生您好，请先拍照留存现场，然后...
           [有温度、有步骤、不慌乱]
```

**底部**：不是万能助手，是**每个岗位一个专业Agent**，各司其职。

## 第8页：越用越聪明（飞轮）

**标题**：这是最核心的——飞轮

**用循环图展示**：

```
    员工日常使用Agent
         ↓
    产生大量真实对话和判断
         ↓
    平台提炼优秀实践 → 封装为新skill
         ↓
    Agent升级，所有用户的Agent都变强
         ↓
    更多人愿意用 → 更多数据 → 更强Agent
         ♻️
```

**一句话**：竞争对手能抄产品，**抄不走这些积累的实践数据**。

---

## 第9页：为什么别人追不上

**标题**：三层护城河

```
第一层  知识飞轮
        用户越多 → 数据越多 → Agent越强 → 时间壁垒

第二层  客户资产绑定
        员工经验沉淀在平台 → 迁移成本极高 → 离不开

第三层  行业壁垒
        保险是重监管重专业的行业 → 通用AI搞不定
```

**一句话**：技术谁都能用，**行业深度 + 数据飞轮**才是我们的壁垒。

---

## 第10页：怎么赚钱

**标题**：商业模式

**简洁展示**：

| 收入来源 | 说明 |
|----------|------|
| 基础版Agent | 按岗位（核保/理赔/客服）收取授权费 |
| 个性化定制 | 员工级别的定制服务费 |
| 持续使用 | Token消耗按用量收费 |

**成本优势**：
- 基础设施 → 客户出（保险公司提供服务器）
- 运维人员 → 客户出钱（我们派人，客户买单）
- **我们的核心成本只有Token和研发**

## 第11页：怎么干

**标题**：第一步

**三步路线图**：

```
第一步（现在）
  选一个岗位打透 → 核保Agent
  做出能用的最小版本

     ↓

第二步
  找一家保险公司做标杆客户
  真实场景跑起来，收集反馈

     ↓

第三步
  有了案例 → 复制到其他岗位（理赔、客服...）
  有了数据 → 飞轮开始转
```

**底部**：不需要一步到位，**先把一个Agent做出来，丢到群里跑**。
