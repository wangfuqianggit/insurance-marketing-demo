# 保险行业AI Agent平台 — 商业模式

---

## 一句话定位

**保险公司的知识蒸馏平台** — 把员工的经验和能力沉淀到AI Agent里，变成公司永不流失的知识资产。

---

## 解决什么问题

| 痛点 | 现状 | 我们的方案 |
|------|------|-----------|
| 老员工离职，经验带走 | 知识在人脑里，人走了就没了 | 经验沉淀在Agent里，人走知识留 |
| 新人培训周期长 | 师傅带徒弟，半年才能上手 | 新人直接用老员工养出来的Agent，加速上手 |
| 各岗位效率低 | 核保/理赔/客服全靠人堆 | 每个岗位有专业Agent辅助，提效 |

---

## 产品形态：AI分身

给保险公司每个员工发一个AI分身，跟着他去任何微信群工作。

### 三步交付

```
① 平台提供基础版Agent（按岗位封装skill）
     ↓
② 员工个性化定制（名字、头像、语言风格）
     ↓
③ Agent跟着员工进入各种群工作
```

### Agent按岗位划分

| Agent | 封装的知识 |
|-------|-----------|
| 核保助手 | 核保规则、健康告知判断、费率计算 |
| 理赔助手 | 理赔流程、定损标准、材料审核 |
| 客服助手 | 产品咨询、保单查询、常见问题 |
| 更多... | 精算、合规、培训等 |

---

## 使用场景：微信群

微信是入口，不需要额外App，打开微信就能用。

### 内部群
```
业务员A + 内勤 + 核保助手 + 理赔助手
→ 内部协作、专业判断、流程讨论
```

### 客户群
```
业务员A + 客户张三 + 客服助手 + 理赔助手
→ 一个群只服务一个客户，隐私安全
→ 多个Agent各司其职，谁专业谁回答
```

### 关键设计
- **一群一客户**：天然解决隐私问题
- **助手跟人走**：业务员把Agent拉到自己需要的群
- **多Agent协作**：一个群里可以有多个Agent，各管各的领域

---

## 飞轮效应：越用越聪明

```
基础版Agent（平台提供skill）
    ↓
员工日常使用
    ↓
平台抽象出优秀实践
    ↓
封装为新skill，升级Agent
    ↓
所有用户的Agent都变强
    ↓
更多用户 → 更多实践 → 更强Agent
    ♻️ 正向循环
```

**核心**：竞争对手能抄产品，抄不走积累的实践数据。

---

## 三层护城河

```
第一层（最强）：知识飞轮
  用户越多 → 数据越多 → Agent越强 → 竞争对手追不上

第二层：客户资产绑定
  员工经验沉淀在平台 → 迁移成本极高 → 换不起

第三层：行业壁垒
  保险是重监管、重专业的行业 → 通用AI公司短期搞不定
```

**核心认知**：技术不是壁垒，行业深度 + 数据飞轮才是。先发优势 = 更多数据 = 更难被追。
