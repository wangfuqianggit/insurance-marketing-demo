## 二、模块逐项拆解

### 模块1：智能拜访全流程管理（优先级：🔴高）

> 核心场景：代理人外出拜访客户的完整闭环

#### 拜访前 | 拜访中 | 拜访后

| 阶段 | 功能点 | 说明 | 清晰度 |
|------|--------|------|--------|
| **拜访前** | 客户优先级排序与锁定 | 基于知客系统的客户画像，智能推荐拜访顺序 | ✅ 清晰 |
| | 行程路线智能规划 | 客户地理位置聚类，最优路线，交通耗时测算 | ✅ 清晰 |
| | 拜访目标与沟通议题设定 | 根据客户经营阶段推荐沟通议题 | ⚠️ 需要明确"客户经营阶段"的定义 |
| | 定制方案与物料准备 | 一键生成建议书+物料清单 | ✅ 清晰 |
| **拜访中** | 语音实时转写 | 多语种识别，解放双手 | ✅ 清晰 |
| | 智能建议书辅助 | 图表展示、多方案对比、动态收益演示 | ✅ 清晰 |
| | 专属话术自动生成 | 基于客户画像+实时对话，生成话术 | ⚠️ 话术生成需要大模型支撑 |
| | 智能问答支撑 | 对接多个知识库 | ❓ 五曦知识库、AI家服务体系是什么？需要客户提供 |
| **拜访后** | 智能小结生成 | 语音记录→关键信息提取→结构化归档 | ✅ 清晰 |
| | 跟进待办自动生成 | 24h/48h标准化跟进动作 | ✅ 清晰 |
| | 数据同步与共享 | 自动同步至CRM | ✅ 清晰 |

**❓待确认：**
- "知客系统"是长城人寿自己的系统还是第三方的？数据接口是否开放？
- 语音转写是用哪家的能力（科大讯飞/阿里/腾讯/自研）？
- 多语种识别具体需要支持哪些语言？（方言？）
- 知识库的内容来源和更新机制？

---

### 模块2：智能化陪访管理（优先级：🔴高）

> 核心场景：资深代理人/主管陪访新人，现场指导

| 功能点 | 说明 | 清晰度 |
|--------|------|--------|
| 陪访对象智能筛选 | 自动识别高潜力/复杂需求客户，匹配陪访清单 | ✅ 清晰 |
| 陪访前期筹备 | 匹配陪访人员、资源排期、物料清单 | ⚠️ 如何定义"匹配资质"？按职级？按险种专长？ |
| 现场协同分工 | 主谈/辅助角色划分，应急预案 | ✅ 清晰 |
| 陪访过程记录 | 实时记录客户需求与反馈，标注亮点与问题 | ✅ 清晰 |
| 陪访效果复盘 | 成单转化率统计、问题归因、短板分析 | ✅ 清晰 |
| 优秀案例沉淀 | 结构化沉淀至共享知识库 | ✅ 清晰 |

**❓待确认：**
- 陪访模块和拜访模块是分开的还是在同一个流程里？（比如：拜访时可以选择"有陪访"模式）
- 陪访人员的排期是自动还是人工确认？

---

### 模块3：智能建议书生成（优先级：🔴高）

> 核心场景：代理人给客户定制保障方案

| 功能点 | 说明 | 清晰度 |
|--------|------|--------|
| FBS销售流程智能化 | 自动评估客户经营阶段，预判保障/理财需求 | ⚠️ "FBS销售流程"是长城人寿自己的方法论吗？需要明确 |
| 客户信息自动填充 | 同步客户基本信息和家庭结构 | ✅ 清晰 |
| 保障额度智能测算 | 根据资产/家庭/已有保障测算缺口 | ❓ 测算规则是什么？需要精算模型还是固定公式？ |
| 方案参数灵活配置 | 保障额度、缴费期限、保障责任可选 | ✅ 清晰 |
| 可视化方案呈现 | 对比图表、收益演示、多方案对比 | ✅ 清晰 |
| 个性化内容定制 | 偏好内容植入，专属话术 | ✅ 清晰 |

**❓待确认：**
- **核心问题**：建议书是HTML/PDF？展示形式是什么？
- 产品库的数据从哪来？长城人寿的产品条款需要接入
- 保额测算的逻辑是简单的规则引擎还是需要精算级别？

## 三、关键依赖与风险清单

| 依赖项 | 依赖方 | 风险等级 |
|--------|--------|----------|
| 知客系统数据接口 | 长城人寿/第三方 | 🔴 核心依赖，没有数据啥都干不了 |
| 语音转写能力 | 需要接入第三方（讯飞/阿里云等） | 🟡 成熟方案，选型即可 |
| 大模型能力 | 需要明确是用哪个模型（GPT/文心/自研） | 🔴 影响架构设计和成本 |
| 产品库/条款数据 | 长城人寿提供 | 🟡 需要结构化整理 |
| 知识库内容（五曦/AI家等） | 长城人寿提供 | 🔴 话术生成和智能问答的核心素材 |
| 合规规则库 | 长城人寿合规部 | 🟡 可以后期建设 |
| FBS销售流程定义 | 长城人寿 | 🟡 纯业务逻辑，确认即可 |

## 四、Demo建议聚焦范围

如果要做演示Demo（给客户看），建议主攻：

**第一优先级（Demo必做）：**
1. **模块1：智能拜访全流程** → 最能体现"AI"价值，故事感最强
2. **模块3：智能建议书生成** → 产生式AI最直观的展示

**第二优先级（Demo可做）：**
3. **模块2：陪访管理** → 展示管理的闭环能力
4. **模块4 + 5：日程+客户经营** → 展示平台完整度

**Demo不做：**
- 模块6（数据同步）→ 后端基建，前端看不到
- 模块7（合规风控）→ 太沉重，不适合初期展示

## 五、下一步建议

1. **跟客户确认核心问题**（见第三部分的依赖项）
2. **确定呈现方式**：Web App / 小程序 / H5
3. **确定技术底座**：用什么大模型、语音能力、地图服务
4. **确定数据对接方案**：知客系统能提供什么数据、接口形式
5. **确定目标用户量级**：影响架构选型
