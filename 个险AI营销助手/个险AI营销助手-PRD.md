# 个险AI营销助手 — 产品需求文档 (PRD)

**版本**: v1.0 (Demo)  
**更新日期**: 2026-07-14  
**产品形态**: 移动端 H5（手机框架 390×844px 模拟）  
**技术栈**: 单 HTML 文件，纯前端（CSS + JavaScript，无外部依赖）

---

## 一、产品定位

面向个险代理人（营销员）的 AI 赋能营销辅助工具。以对话式 AI 为核心交互方式，覆盖代理人日常工作全流程：客户拜访管理、智能建议书、物料管理、路线规划、录音转译、AI 分析总结、陪访申请审批、日程管理等。

---

## 二、核心页面

### 2.1 首页工作台

| 区域 | 内容 |
|------|------|
| 顶部状态栏 | 时间、信号 |
| 头部 | AI 图标、应用名称「个险AI营销助手」、AI 状态指示器（绿色脉冲点 + "AI就绪"） |
| 洞察栏 | 渐变深色背景，显示"早上好小王！今日N位已约拜访 · 客户名 时间"，点击可查看摘要 |
| 今日已约拜访 | 卡片列表，每行：客户头像（首字母+渐变背景）、姓名、时间、陪访标签、地址；操作按钮：拜访、筹备、↻（替换/新增）|
| 明日待访 | 明日预安排客户的准备列表 |
| 已预约下次拜访 | 已完成拜访后预约的下次时间 |
| 本周效能 | 柱状图（拜访量）+ 统计数据（拜访数、成单数、保费） |
| 底部输入区 | ChatGPT 风格快捷指令 + 按住说话语音按钮 + 文字输入框 |

### 2.2 AI 对话页（spChat）

首页文字输入或语音输入后进入对话页面：
- 聊天气泡样式（用户蓝色右对齐，AI 灰色左对齐）
- 每条消息带时间戳
- 底部支持文字输入和按住说话
- AI 会根据输入内容自动回复

### 2.3 拜访流程

#### 2.3.1 拜访前准备（筹备弹窗）
每个客户可查看筹备清单，AI 建议准备内容，包括：
- 建议书就绪状态
- 物料就绪状态  
- 沟通要点提示
- 底部快捷按钮：建议书、物料

#### 2.3.2 拜访中（录音+转译）
点击"拜访" → 进入拜访详情页：
- 客户信息展示（姓名、地址、年龄、标签）
- 拜访前准备清单（已准备/建议准备）
- AI 建议提示
- **点击"开始录音"** → 进入录音模式：
  - 录音状态栏（深色渐变底 + 红色脉冲点 + 计时 + 声波动画）
  - 对话转录区：聊天气泡式展示，代理人与客户分列左右，带头像和时间戳
  - AI 实时分析卡片，显示正在分析的要点
  - 结束录音按钮

#### 2.3.3 拜访后（AI 总结）
录音结束 → AI 自动分析生成拜访小结：
- 对话摘要
- 关键结论和客户意向
- 自动生成的跟进待办（今天、明天、后续）
- 预约下次拜访时间（日期选择器 + 时间选择器，支持智能识别自然语言时间）
- 发送感谢消息

### 2.4 建议书（spProposal）

每个客户可生成个性化保险方案建议书：
- AI 需求诊断（基于客户画像）
- 推荐方案详情（产品名、保额、保费、保障范围等）
- 投保详情表格
- AI 促成建议
- 发送给客户、导出 PDF

### 2.5 物料清单（spMaterials）

客户拜访所需物料管理：
- 物料列表含状态：✅ 已生成 / ⏳ 生成中 / 🖨️ 待打印
- 每个物料可操作（如打印）

### 2.6 路线规划（spRoute）

- 动态模拟地图（CSS 绘制，含道路网格、绿地）
- 每位客户在地图上有固定位置标记
- 路线列表显示拜访顺序，每站附带距离和交通时间
- 每站可 ▲▼ 调整顺序，调整后自动重算时间和路线
- 总里程、推荐出发时间
- 新增拜访客户按钮
- 确认路线按钮

### 2.7 陪访管理（spCoach）

双端设计，支持成员/主管切换：

**成员端**：
- 查看已提交的陪访申请及状态（待审批/已通过）
- 发起新陪访申请（选择客户、选择主管、填写理由）
- 已通过申请的拜访自动展示在日程中

**主管端**：
- 查看待审批申请列表（含申请人、客户、理由、时间）
- 同意/拒绝操作
- 已审批记录

### 2.8 日程管理（spSchedule）

- 日视图时间线展示
- 支持切换日期（今日 / 明日 / 后续）
- 顶部周栏显示当前日期所在周
- 展示当天所有拜访时间线
- 后续预约列表

## 四、数据模型

### 客户（C）
```javascript
{
  name: { a: '首字母', bg: '头像渐变背景', tag: '标签', addr: '地址',
         age: '年龄', fam: '家庭状况', income: '年收入', prem: '保费',
         score: '评分', policies: '保单数', gap: '保障缺口', note: 'AI建议' }
}
```

### 拜访（visits）
```javascript
{
  today: [{ name, time, status }],
  tomorrow: [{ name, time, status }]
}
```

### 陪访申请（coachApps）
```javascript
{
  id, customer, manager, applicant, reason, time, status: 'pending'|'approved'|'rejected'
}
```

### 对话消息（chatMessages）
```javascript
[{ speaker: 'user'|'ai', text, time }]
```

### 拜访录音（visitTranscript）
```javascript
[{ speaker: 'agent'|'customer', text, time }]
```

## 五、交互规范

| 交互 | 说明 |
|------|------|
| 页面切换 | 子页面从右侧滑入（translateX），0.3s ease |
| 底部弹出 | picker 从底部滑出（slideUp） |

## 六、视觉设计

- **配色**: 主色 `#2563eb`（蓝），辅色 `#8b5cf6`（紫），强调色 `#f59e0b`（橙）
- **圆角**: 大卡片 16px，小卡片 10px，按钮 7px
- **阴影**: 双层阴影 `0 1px 4px rgba(0,0,0,0.04), 0 4px 16px rgba(0,0,0,0.04)`
- **字体**: 系统字体栈 `-apple-system, PingFang SC, Helvetica Neue`
- **渐变**: 头部 AI 图标、洞察栏、录音条使用渐变背景
- **模拟手机**: 390×844px 圆角容器，带刘海和状态栏

## 八、局限性（Demo 阶段）

1. 所有数据为静态模拟，无后端 API
2. 录音为模拟演示，非真实语音识别
3. AI 分析内容为预设占位文本
4. 地图为 CSS 模拟绘制，非真实地图数据
5. 无用户登录/鉴权系统
6. 无数据持久化（刷新即重置）
