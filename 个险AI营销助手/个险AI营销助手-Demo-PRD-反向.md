# 个险AI营销助手 — Demo 反向 PRD

> 本文档基于 Demo HTML（20260714 版）反向梳理，完整覆盖 Demo 中所有功能和交互细节。
>
> **产品形态**: 移动端 H5（390×844px，模拟 iPhone 尺寸）
> **技术栈**: 单 HTML 文件，纯前端，无后端依赖

---

## 一、产品定位

代理人专属的 AI 营销全能助手，覆盖"见面前→见面中→见面后→日常经营"全场景。同时支持**成员端**和**主管端**双视角。

---

## 二、核心架构

### 2.1 双视角切换

| 视角 | 适用角色 | 特点 |
|------|---------|------|
| **成员端**（默认） | 代理人（如王小明） | 个人拜访管理、陪访申请、个人效能看板 |
| **主管端** | 团队主管（如李国强） | 团队日程、陪访审批、团队效能看板 |

- 顶部 Tab 切换，切换后首页和各子页面内容联动变化
- 状态栏显示当前视角标签

### 2.2 页面导航

- **首页**（工作台）→ 默认首页
- **8 个子页面**：智能建议书、物料清单、拜访小结、路线规划、陪访管理、日程管理、拜访中、AI 对话、内容营销
- **底部浮层**：快捷指令栏 + 语音/文字输入区
- **底部弹出选择器**：客户选择、筹备菜单、预约时间

## 三、功能模块

### 模块 1：首页工作台

#### 成员端首页

| 区域 | 内容 |
|------|------|
| **AI 欢迎栏** | 问候语 + 待审批陪访数量，点击进入陪访管理 |
| **今日已约拜访** | 拜访列表，每位显示：头像/姓名/时间/地址/陪访标签，操作：拜访、筹备、更换 |
| **快捷按钮** | 查看日程、规划路线 |
| **本周效能** | 周一至周五柱状图（拜访量）、汇总数据：拜访数/成单数/保费 |

| **后续已预约** | 已预约的拜访列表 |

**交互：**
- 点击客户行 → 进入拜访页面

- 点击"查看日程" → 进入日程管理
- 点击"规划路线" → 进入路线规划
- 点击效能区域 → 进入日程管理

| 区域 | 内容 |

### 模块 2：智能拜访全流程

#### 2.1 拜访前筹备

点击"筹备"按钮弹出底部菜单，包含 AI 建议 + 三项筹备条目：

**AI 建议栏：** 根据客户画像生成针对性的拜访策略建议

**筹备条目清单（每个客户不同）：**
| 条目 | 说明 |
|------|------|
| 📄 智能建议书 | 已生成/建议准备 |
| 📦 拜访物料 | 已就绪/建议准备 |
| 🗺️ 路线规划 | 建议规划最优路线 |
| 🤝 陪访申请 | 高价值客户建议申请陪访 |
| 其他按客户定制 | 如保单到期提醒函、需求分析问卷、入门资料包等 |

**筹备条目的状态：**
- ✅ 已准备（绿色）
- ⏳ 建议准备（红色/橙色）

**底部操作按钮：**
- 📄 建议书 → 打开建议书页面
- 📦 物料 → 打开物料清单页面

#### 2.2 拜访中

**拜访前页面：**

| 区域 | 内容 |
|------|------|
| 客户信息头 | 客户姓名、地址、年龄、标签 |
| 拜访前准备 | 建议书状态、物料状态、陪访分工、沟通要点 |
| AI 建议 | 基于客户画像的智能建议 |
| 开始按钮 | "开始录音 · AI 实时转录"按钮 |

**陪访协作：**
- 陪访分"代理人主谈"和"主管主谈"两种模式
- 主谈人专注沟通，协助人负责录音和记录
- 协助人可实时添加观察记录

**录音中页面：**

| 区域 | 内容 |
|------|------|
| 录音状态栏 | 红色脉冲圆点 + 客户名 + 计时器 + 声波动画 |
| 对话转录区 | 对话气泡，区分代理人（蓝色）和客户（绿色），显示发言时间 |
| AI 实时分析 | 实时识别关键话题、客户情绪、推进建议 |
| 实时记录（协助人） | 协助人可输入观察笔记 |
| 结束按钮 | "结束录音 · AI 自动分析"按钮 |

#### 2.3 拜访后 AI 小结

| 区域 | 内容 |
|------|------|
| 完成状态 | ✅ 图标 + 客户名 + "拜访已结束" |
| **拜访摘要** | 客户、地址、沟通主题、核心内容 |
| **AI 对话分析** | 对话时长、关键话题数、客户意向、关键洞察、客户情绪评级 |
| **客户需求识别** | 需求标签（如重疾续保、配偶保障、生日礼遇等） |
| **客户意向评估** | 意向等级（如 B+）、签单概率进度条、预计保费范围 |
| **跟进待办** | 自动生成的待办列表（可编辑、可切换完成状态） |
| **预约下次拜访** | 按钮 → 弹出预约时间选择器 |

**待办系统：**
- 拜访结束后自动生成 2-3 条跟进待办
- 待办可点击编辑文字，可点击方框切换完成状态
- 待办带截止日期标签（今天/明天/具体日期）
- 待办在日程管理页面同步展示

### 模块 3：智能建议书

**信息头：** 客户姓名、年龄、家庭状况、年收入、年保费、已有保单、健康评分

| 区域 | 内容 |
|------|------|

| 🏆 推荐方案 | 产品名称、保额、保障期限、缴费期限、年缴保费、保障范围、轻症豁免等 |
| 🎯 AI 促成建议 | 针对性的促成话术和建议 |
| 操作按钮 | 📤 发送给客户、📥 导出 PDF |

**数据来源：**
- 客户数据：姓名、年龄、家庭、收入、保费、保单数、评分、保障缺口、备注
- 每位客户数据不同，Demo 内置 7 位客户数据

### 模块 4：物料清单

**AI 分析头：** 基于客户画像自动识别本次拜访核心议题，推荐物料

**物料列表（每位客户不同）：**
| 物料名 | 状态 |
|--------|------|
| 建议书/方案书 | ✅ 已生成 / ⏳ 生成中 |
| 客户贺卡/礼品 | ✅ 已生成 |
| 产品单页/说明 | ✅ 已生成 |
| 分析报告 | ⏳ 生成中 |
| 投保申请书 | 🖨️ 待打印 |

**物料状态类型：**
- ✅ 已生成（绿色）
- ⏳ 生成中（橙色，带进度条动画）
- 🖨️ 待打印（蓝色，可点击打印）

**统计：** X 项已就绪 · Y 项生成中 · Z 项待打印

**操作按钮：** 📦 一键打包、🖨️ 打印全部

### 模块 5：路线规划

**模拟地图：**
- 公司出发点标记
- 每位客户的固定位置标记（带编号 1、2、3...）
- 显示客户姓名标签

**路线列表（可调序）：**
- 公司（出发）→ 第 1 站 → 第 2 站 → 第 3 站...
- 每站间显示：↓ Xkm · 约 Ymin
- 每站显示序号、客户名、拜访时间
- **↑↓ 按钮**可调整拜访顺序，调整后时间自动重算

**信息汇总：**
- ⏱️ 时间段（如 09:00→12:00）
- 总里程（如 35km）
- 推荐出发时间（如 08:23）

**操作按钮：**
- ✅ 确认路线（确认后隐藏排序按钮）
- ✏️ 修改路线（重新显示排序按钮）

**算法：** 基于坐标的 TSP 近似解，自动计算行程时间和耗时

### 模块 10：预约下次拜访

**AI 智能识别：**
- 从对话转录中自动识别时间信息（明天/后天/下周/具体时间）
- 显示 AI 识别结果及建议时间

**快速选择：**
- 日期快速选择：明天、后天、下周一、下周三、下周
- 时间快速选择：9:00、10:00、14:00、15:00

**手动选择：**
- 日期选择器（date input）
- 时间选择器（time input）

**确认后：**
- 预约记录加入"后续已预约"列表
- 如预约的是明天，自动加入明日拜访列表

### 模块 11：待办事项系统

**系统：**
- 全局待办列表，每个待办有：id、关联客户、文字内容、截止日期、来源（拜访/手动）
- 拜访结束后自动生成待办
- 待办在日程管理、拜访小结等多处同步展示

**交互：**
- 点击方框：切换完成/未完成
- 点击文字：进入编辑模式（inline editing）
- 点击 ✕：删除待办

**截止日期类型：**
- 今天（红色）
- 明天（橙色）
- X 天后（绿色）
- X 月 X 日（绿色）

---

### 模块 12：客户更换与新增

**更换客户：**
- 从今日拜访中点击"↻"按钮
- 弹出备选客户列表（预置的 reserveCustomers）
- 选择后替换原客户，路线自动重算

**新增拜访：**
- 在日程管理的空时段点击"➕ 点击新增"
- 弹出备选客户列表
- 选择后加入今日拜访，路线自动重算

## 六、交互规范

### 页面切换
- 子页面从右侧滑入（slide-in）
- 返回时从左侧滑出
- 首页卡片向上淡入动画

### 手势交互
- 语音按钮：按住说话（mousedown/touchstart）、松开发送（mouseup/touchend）
- 按钮按压：active 状态缩小 0.85-0.93 缩放

### 反馈
- Toast 提示：底部居中，1.8 秒自动消失
- 录音脉冲动画
- 声波动画（AI 实时转录中）
- 加载进度条动画（物料生成中）

### 响应式
- 390px 以上：居中显示带圆角的手机外壳
- 420px 以下：全屏显示，无圆角

## 七、功能边界说明（Demo 限制）

1. **语音输入** — Demo 中仅模拟 UI 交互，无真实语音识别
2. **AI 对话** — Demo 中为静态模拟数据，无真实 LLM 调用
3. **地图** — 静态模拟地图，非真实地图服务
4. **数据** — 所有数据硬编码（客户、拜访、待办等），无后端持久化
5. **内容营销** — 海报和文章为预置数据，非 AI 实时生成
6. **路线算法** — 简化的坐标距离计算，非真实路径规划
