## 模块1：智能拜访全流程管理

### 1.1 客户优先级排序与锁定

**用户交互：**
- 打开拜访模块，看到今日推荐拜访客户列表（卡片式，按优先级排序）
- 每个卡片显示：客户名、关系强度、预估价值、上次拜访时间
- 点击"锁定"确认今日拜访，锁定后进入路线规划

**技术实现：**

```
知客系统 ──→ 客户画像数据 ──→ 排序算法 ──→ 推荐列表 ──→ 代理人确认
     │            │               │
     │       ┌─────┴─────┐        │
     │       │ 客户价值分   │        │
     │       │ 距上次拜访   │        │
     │       │ 近期互动     │        │
     │       │ 保单到期     │        │
     └───────┴───────────┘────────┘
```

- **排序规则引擎**：综合以下维度加权打分
  - 客户价值分（来自知客系统）— 权重40%
  - 距上次拜访天数 — 权重25%（越久越优先）
  - 近期互动频率 — 权重20%（互动少优先）
  - 保单即将到期/生日等事件 — 权重15%
- **打分公式**：`总分 = 价值分×0.4 + 拜访紧迫度×0.25 + 互动衰减度×0.2 + 事件分×0.15`
- **实现方式**：后端定时任务（每日凌晨计算），结果写入Redis缓存，早上打开时秒出

### 1.2 行程路线智能规划

**用户交互：**
- 锁定多个客户后，点击"规划路线"，自动生成最优拜访顺序
- 地图上标注客户位置，显示路线和预计总耗时
- 支持手动拖拽调整顺序

**技术实现：**
- 调用**高德地图/腾讯地图路径规划API**
- 输入：多个客户地址坐标
- 算法：**旅行商问题（TSP）近似解** - 使用贪心最近邻算法
  - 以代理人当前位置为起点
  - 每次选择最近的未拜访客户作为下一站
  - 最终回到公司或回家（可选）
- 输出：有序路线 + 每段路程耗时 + 总耗时
- 前端展示：地图组件（高德JS API / 腾讯地图SDK）

### 1.3 语音实时转写

**用户交互：**
- 点击"开始录音"，手机开始拾音
- 实时显示转写文字（类似语音输入法）
- 对话中可手动标记关键节点（"客户提到需求"、"异议"、"承诺"等）
- 结束时自动生成结构化记录

**技术实现：**

```
手机麦克风 ──→ PCM音频流 ──→ 语音转写API ──→ 文字流 ──→ 前端展示
                              │
                     ┌────────┴────────┐
                     │ 实时转写（流式）    │
                     │ 阿里云/讯飞实时转写 │
                     │ WebSocket传输    │
                     └─────────────────┘
                              │
                    ┌─────────┴──────────┐
                    │ 关键信息实时提取      │
                    │ （后端流式处理）      │
                    └────────────────────┘
```

- **语音转写**：接入**阿里云实时语音转写**或**讯飞语音转写**
  - 流式接口，WebSocket连接
  - 支持普通话+方言（可选）
  - 返回带时间戳的文字片段
- **关键信息提取**：文字流实时送入LLM，按15-30秒间隔做轻量分析
  - LLM Prompt示例：
  ```
  你是一个保险销售助理，正在实时分析对话内容。
  请从以下对话片段中提取：
  1. 客户提到的需求关键词
  2. 客户提出的异议/顾虑
  3. 代理人的应答要点
  4. 是否有成交信号

  对话内容：[实时文字]
  ```
- **技术要点**：需要做语音活动检测（VAD），过滤静音时段，节省API费用

### 1.4 专属话术自动生成

**用户交互：**
- 拜访中遇到客户提问或异议，点击"话术助手"
- 输入问题或选择场景（如"客户说太贵了"、"客户要对比别家"）
- 系统基于客户画像+当前对话上下文，生成专属话术
- 话术展示为卡片，可一键复制

**技术实现：**
- **RAG架构（检索增强生成）**

```
客户画像 + 对话上下文 + 问题
           │
           ▼
    ┌──────────────┐
    │ 向量检索      │ ← 知识库（产品FAQ、话术库、异议处理库）
    │ 检索相似案例  │
    └──────┬───────┘
           │ 检索到的相关话术片段
           ▼
    ┌──────────────┐
    │ LLM生成话术   │ ← Prompt：结合客户画像+检索结果+当前问题
    └──────┬───────┘
           │
           ▼
       话术卡片（展示）
```

- **知识库构建**：
  - 产品知识库：条款、费率、投保规则 → 向量化（embedding）存入向量数据库
  - 话术库：优秀话术模板 → 按场景分类（异议处理、促成、约访等）
  - 合规规则库：不能说的内容 → 用于输出过滤
- **LLM调用**：通义千问/DeepSeek
  - Prompt模板：
  ```
  你是一个资深的保险销售专家。
  客户信息：{年龄、家庭结构、已购保单、关注点}
  当前对话背景：{上下文摘要}
  客户提出：{问题}

  请生成一段应对话术，要求：
  1. 通俗易懂，适合口语表达
  2. 结合客户实际情况（个性化）
  3. 符合监管合规要求（不夸大、不误导）
  4. 给出"推荐说"和"避免说"两个版本
  ```

### 1.5 智能小结生成

**用户交互：**
- 拜访结束，点击"生成小结"
- 系统自动基于语音记录生成结构化拜访记录
- 包括：沟通内容摘要、客户需求、客户异议、下一步计划
- 代理人可编辑修改后确认保存

**技术实现：**
```
完整对话文本
     │
     ▼
┌──────────────┐
│  LLM总结      │ ← Prompt：结构化提取信息
│  （一次性处理） │
└──────┬───────┘
     │
     ▼
┌──────────────────┐
│ 输出结构化JSON    │
├──────────────────┤
│ {                │
│   拜访时间: "",   │
│   客户: "",      │
│   沟通主题: "",   │
│   客户需求: [],   │
│   客户异议: [],   │
│   承诺事项: [],   │
│   下一步计划: "", │
│   综合评估: ""   │
│ }                │
└──────────────────┘
     │
     ▼
  写入数据库 + 同步知客系统
```

## 模块2：智能化陪访管理

### 2.1 陪访对象智能筛选

**技术实现：**
- 规则引擎+评分模型
- 筛选条件：
  - 客户价值分 > 阈值（高价值客户需要陪访）
  - 需求复杂度（涉及多个险种/大额保单）
  - 代理人经验值（新人/新产品的单子优先陪访）
  - 历史成单转化率（之前搞不定的客户，派高手去）
- 生成陪访推荐清单，推送给团队主管

### 2.2 陪访人员匹配

**技术实现：**
- 匹配算法：分析陪访人员的专长（重疾/理财/团险）、职级、空闲时间
- 类似"推荐系统"思路：`match_score = 专长匹配度×0.4 + 经验等级×0.3 + 时间可调度×0.3`
- 生成推荐排名，主管确认或手动调整

### 2.3 陪访效果复盘

**技术实现：**
- **LLM分析**：根据陪访记录+成单结果，自动生成复盘报告
- Prompt模板：
```
你是一个保险销售教练。
以下是代理人的陪访记录和结果：
{陪访记录}

请分析：
1. 本次陪访中做得好的是什么
2. 存在什么问题/改进空间
3. 该代理人的短板是什么（产品知识？沟通技巧？促成能力？）
4. 给出具体的提升建议
```
- 复盘报告自动归档至代理人成长档案

### 2.4 优秀案例结构化沉淀

**技术实现：**
- 将成功的陪访案例转化为结构化知识条目
- LLM提取：客户类型、需求场景、处理策略、关键话术、结果
- 存入共享知识库，支持按标签/关键词检索
- 后续其他代理人遇到类似场景，可智能推荐参考案例

## 模块3：智能建议书生成

### 3.1 FBS销售流程智能化

**技术实现：**
- FBS流程引擎：客户经营阶段识别（潜在/意向/成交/复购/转介绍）
- 每个阶段预定义标准化动作和话术
- 根据客户数据自动判断当前阶段

### 3.2 保障需求预判与额度测算

**技术实现：**

```
客户信息（年龄/收入/家庭/已有保单）
     │
     ▼
┌──────────────┐
│ 需求预判规则   │
│              │
│ • 家庭责任    │ → 寿险缺口 = 房贷+子女教育+父母赡养 - 已有寿险
│ • 医疗覆盖    │ → 重疾缺口 = 治疗费用+收入损失 - 已有重疾
│ • 养老储备    │ → 养老缺口 = 预期养老费用 - 已有养老金
│ • 财富传承    │ → 根据资产规模判断需求
└──────┬───────┘
     │
     ▼
┌──────────────┐
│ LLM生成方案    │
│ 说明：用大白话 │
│ 解释为什么需要 │

```

### 3.3 可视化方案呈现

**技术实现：**
- 前端使用 ECharts / AntV 渲染图表
- 对比图：不同产品保障责任对比（柱状图）
- 收益演示：分红/年金领取演示（折线图/表格）
- 保障全景图：用图示展示"生病拿钱、老了领钱、走了赔钱"的覆盖
- 支持一键导出为PDF（使用html2pdf或服务端Puppeteer渲染）

### 3.4 AI方案说明生成

**技术实现：**
- LLM根据测算结果生成方案说明文字
- Prompt：
```
你是一个保险顾问，正在为客户解释保障方案。

客户信息：{客户信息}
推荐方案：{产品+保额+保费}

请生成一段通俗易懂的方案说明，包括：
1. 这个方案解决了什么风险
2. 为什么这样搭配
3. 每年交多少、交多久、保什么
4. 用比喻的方式让客户一听就懂
```
- 输出经合规过滤后展示

## 模块4：日程智能管家

### 4.1 动态日程调整

**技术实现：**
- 基于时间槽的日程数据结构
- 冲突检测算法：新插入事件与已有事件的时间段重叠检测
- 冲突化解策略：
  - 按优先级覆盖（拜访 > 会议 > 培训 > 其他）
  - 智能建议：推迟低优先级的，或推荐新的空闲时间段
- 突发任务插队：人工确认后自动调整后续安排

### 4.2 日程效能分析

**技术实现：**
- 统计维度：拜访时间、交通时间、等待时间、非业务时间
- 公式：`效能分 = 有效拜访时间 / 总工作时间 × 100%`
- 每周自动生成分析报告，给出优化建议
- 报告也是LLM生成的，类似周报总结

## 模块5：客户智能化经营

### 5.1 AI素材自动生成

**技术实现：**
- 预设模板+LLM填充

| 素材类型 | 模板 | LLM参与度 |
|----------|------|-----------|
| 朋友圈文案 | 模板：今日拜访感悟 + 客户故事 + 生活感悟 | 中（填入个性化内容） |
| 营销海报 | 固定模板+客户名 | 低（主要是设计排版） |
| 生日祝福 | 模板：称呼+祝福语+签名 | 低 |
| 话题文章 | LLM从素材库生成一篇短文 | 高 |
| 邀约话术 | 模板：问候+理由+邀约时间 | 中 |

- 用户选择素材类型 → 选择模板 → AI生成 → 用户编辑 → 发布/发送

### 5.2 客户分层经营

**技术实现：**
- 分层标准（示例，可配置）：

| 层级 | 标准 | 经营策略 |
|------|------|----------|
| A-高价值 | 年保费>10万或资产>100万 | 每月拜访+定制方案 |
| B-中价值 | 年保费1-10万 | 每季度拜访+节日维护 |
| C-潜力 | 新客户/低保费 | 按计划培养 |
| D-沉默 | 超过1年未互动 | 激活策略 |

- 系统自动推荐每个层级的标准经营动作

## 模块6：数据同步与协同

### 技术实现方案

```
┌──────────────┐         ┌──────────────┐
│  知客系统     │ ←─API──│   AI营销助手  │
│  (第三方厂商)  │         │   (我们)      │

```

- **同步策略**：
  - 增量同步（每天凌晨）+ 实时查询（双写）
  - 冲突解决：以最近更新为准
- **接口形式**：RESTful API / GraphQL
- **安全**：接口鉴权（AK/SK）+ 数据加密传输

## 模块7：合规与风控管理

### 技术实现

- **两层检测机制**：

```
第一层：规则引擎（快）
┌─────────────────────┐
│ 关键词黑名单         │ → "保证""肯定""100%"等违规词
│ 正则表达式           │ → 收益承诺、比同类产品等模式
└─────────────────────┘
         │ 命中 → 拦截/警告
         ▼
第二层：AI辅助判断（准）
┌─────────────────────┐
│ LLM语义理解          │ → "这个产品比银行存款划算"
│ 上下文分析           │ → 结合上下文判断是否违规
└─────────────────────┘
```

- 规则可配置：合规人员可通过后台添加/修改规则
- 检测时机：话术生成时同步检测、外发资料提交前检测

## 附录：LLM使用全景

| 场景 | 调用时机 | 输入 | 输出 | 实时性要求 |
|------|----------|------|------|-----------|
| 话术生成 | 代理人点击"话术助手" | 客户画像+问题 | 话术卡片 | 高（秒级） |
| 智能问答 | 代理人提问 | 问题 | 答案 | 高（秒级） |
| 拜访小结 | 拜访结束 | 对话全文 | 结构化记录 | 中（秒级） |
| 建议书说明 | 生成建议书时 | 方案数据 | 说明文字 | 中（秒级） |
| 复盘分析 | 陪访结束 | 陪访记录 | 复盘报告 | 低（可异步） |
| 案例萃取 | 成单后 | 成功案例 | 结构化知识 | 低（异步） |
| 素材生成 | 代理人请求 | 模板+数据 | 文案/内容 | 中（秒级） |
| 合规检测 | 内容提交前 | 要检测的文本 | 合规判定 | 高（秒级） |

### 成本估算参考

- 每次LLM调用平均消耗：约500-1500 tokens
- 单代理人日均调用：约20-50次
- 以1000代理人为例，月均token消耗约：300万-7500万
- 模型选DeepSeek/通义千问，月成本约几百到几千元
